# 医生自动排班功能 — 实现说明与前后端对接文档

> 适用模块：`registration-service`（微服务架构）  
> 功能概要：**排班模板（周规律） + 一键批量生成 + 手动改/停诊 + 7 天放号窗口 + Redis 号源同步**  
> 编写时间：2026-07-05

---

## 1. 功能设计说明

### 1.1 业务模型（两层）

| 层级 | 存储表 | 含义 |
|------|--------|------|
| **模板层** | `doctor_schedule_template` | 描述医生「每周几 + 时段」的常规出诊规律 |
| **实例层** | `doctor_schedule`（原有） | 某一具体日期的可挂号排班，供患者查询/抢号 |

关系：

```text
排班模板（周规律）
    ↓  POST /api/schedules/generate
具体排班实例（doctor_schedule）
    ↓  管理员可 PUT / close 手动调整
患者 GET /api/schedules?bookableOnly=true 查询
    ↓  POST /api/registrations/quick 抢号
```

### 1.2 自动 + 手动如何共存

- **自动**：按模板批量写入 `doctor_schedule`；若「同医生 + 同日期 + 同时段」已存在则 **跳过**（`skippedCount++`），不报错。
- **手动**：原有单条创建/修改/停诊接口保留，优先级高于模板；可对自动生成的实例单独改号源、停诊。
- **放号窗口**：配置 `app.registration.advance-days=7`，患者只能预约「今天起共 7 个自然日」内的排班。

### 1.3 时段枚举

与原有系统一致，常用值：

| timeSlot | 含义 |
|----------|------|
| `MORNING` | 上午 |
| `AFTERNOON` | 下午 |

### 1.4 dayOfWeek 约定（ISO-8601）

| 值 | 星期 |
|----|------|
| 1 | 周一 |
| 2 | 周二 |
| 3 | 周三 |
| 4 | 周四 |
| 5 | 周五 |
| 6 | 周六 |
| 7 | 周日 |

## 2. 数据库变更

### 2.1 新增表

**脚本路径**：`infra/postgres/init/004-schedule-template.sql`

```sql
CREATE TABLE IF NOT EXISTS doctor_schedule_template (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    doctor_id BIGINT NOT NULL,
    department_id BIGINT NOT NULL,
    day_of_week INTEGER NOT NULL,          -- 1=周一 ... 7=周日
    time_slot VARCHAR(16) NOT NULL,        -- MORNING / AFTERNOON
    source_count INTEGER NOT NULL,         -- 号源总数
    fee_amount NUMERIC(10, 2) NOT NULL,    -- 挂号费
    source_type VARCHAR(32),               -- NORMAL / EXPERT 等
    status VARCHAR(32) NOT NULL DEFAULT 'ENABLED',
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted BOOLEAN NOT NULL DEFAULT FALSE,
    CONSTRAINT uk_schedule_template_doctor_day_slot UNIQUE (doctor_id, day_of_week, time_slot)
);
```

**部署注意**：新环境必须先执行该 SQL；已有远程库也需手动执行一次（否则接口报 `relation "doctor_schedule_template" does not exist`）。

### 2.2 原有表（无结构变更）

`doctor_schedule` 表结构不变，自动生成的记录与手动创建写入同一张表。

## 3. 配置项

**文件**：`registration-service/src/main/resources/application.yml`

```yaml
app:
  registration:
    advance-days: 7                    # 患者可预约天数（含当天）
    auto-generate-enabled: false       # 是否启用定时自动生成
    auto-generate-cron: "0 0 23 * * SUN"  # 默认每周日 23:00
```

| 配置项 | 说明 |
|--------|------|
| `advance-days` | 生成默认天数 & 患者 `bookableOnly` 窗口 & 抢号日期校验 |
| `auto-generate-enabled` | `true` 时由定时任务自动调用生成逻辑 |
| `auto-generate-cron` | Spring Cron 表达式 |

## 5. REST API 说明

> 统一前缀：经网关 `http://{host}:10010/api` 或直连 `registration-service` `http://{host}:10023/api`  
> 鉴权：管理端接口需角色 `ADMIN` / `MANAGEMENT` / `REGISTRATION_CLERK`  
> 响应格式：`Result<T>` → `{ code, message, data, timestamp }`

### 5.1 排班模板 API（管理端）

**Base**：`/api/schedule-templates`

#### 创建模板

```http
POST /api/schedule-templates
Authorization: Bearer {token}
Content-Type: application/json

{
  "doctorId": 9201,
  "departmentId": 9101,
  "dayOfWeek": 1,
  "timeSlot": "MORNING",
  "sourceCount": 20,
  "feeAmount": 30.00,
  "sourceType": "NORMAL"
}
```

失败情况：同医生+星期+时段重复 → `500` / message「该医生在此星期时段已有排班模板」

#### 模板列表

```http
GET /api/schedule-templates?pageNo=1&pageSize=10&doctorId=9201&departmentId=9101
```

#### 模板详情

```http
GET /api/schedule-templates/{id}
```

#### 更新模板

```http
PUT /api/schedule-templates/{id}
Content-Type: application/json

{
  "sourceCount": 25,
  "feeAmount": 35.00,
  "sourceType": "EXPERT",
  "status": "ENABLED"
}
```

> 注意：改模板**不会** retroactive 更新已生成的 `doctor_schedule` 实例，只影响后续 generate。

#### 停用模板

```http
POST /api/schedule-templates/{id}/disable
```

### 5.2 批量生成 API（管理端）

```http
POST /api/schedules/generate
Authorization: Bearer {token}
Content-Type: application/json

{
  "days": 7,
  "startDate": "2026-07-05",
  "doctorId": 9201,
  "departmentId": 9101
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `days` | 否 | 默认取 `advance-days`（7） |
| `startDate` | 否 | 默认今天 |
| `doctorId` | 否 | 只针对某医生模板生成 |
| `departmentId` | 否 | 只针对某科室模板生成 |

**响应 data 示例**：

```json
{
  "startDate": "2026-07-05",
  "endDate": "2026-07-11",
  "createdCount": 3,
  "skippedCount": 1,
  "createdScheduleIds": [61, 62, 63]
}
```

### 5.3 排班查询 API（管理端 + 患者端）

```http
GET /api/schedules?pageNo=1&pageSize=10&doctorId=9201&departmentId=9101&scheduleDate=2026-07-05&timeSlot=MORNING&bookableOnly=false
```

| 参数 | 说明 |
|------|------|
| `bookableOnly=false` | 管理端：查全部（默认） |
| `bookableOnly=true` | **患者端必传**：仅返回今天起 7 天内、ENABLED、有剩余号源的排班 |

### 5.4 抢号（患者端，已有改动）

```http
POST /api/registrations/quick
Authorization: Bearer {patientToken}
Content-Type: application/json

{ "scheduleId": 61 }
```

- **不要传** `patientId`（后端从 token 取）
- 若排班日期超出 7 天窗口 → `400`「该排班不在可预约时间窗口内」

### 5.5 手动调整单条排班（管理端）

> 「暂停一次排班」= 对**某一天的具体排班实例**停诊，**不影响**周模板，下次 generate 仍可能再生成（若该日实例已被跳过则保持现状）。

#### 修改号源 / 费用 / 类型

```http
PUT /api/schedules/{id}
Authorization: Bearer {token}
Content-Type: application/json

{
  "sourceCount": 25,
  "feeAmount": 55.00,
  "sourceType": "EXPERT"
}
```

| 字段 | 说明 |
|------|------|
| `sourceCount` | 总号源；不能小于已挂出数量（`sourceCount - availableCount`） |
| `feeAmount` | 挂号费 |
| `sourceType` | `NORMAL` / `EXPERT` 等 |
| `status` | 也可传 `DISABLED` / `CLOSED`（推荐停诊用下方专用接口） |

#### 暂停 / 停诊单次排班（推荐）

```http
POST /api/schedules/{id}/close
Authorization: Bearer {token}
```

- 将 `doctor_schedule.status` 置为 **`CLOSED`**
- 患者端 `bookableOnly=true` **不会返回**该条（仅 `ENABLED` 且有余号）
- **模板不变**：只停这一次实例；如需永久取消某 weekday 规律，应 `POST /api/schedule-templates/{id}/disable`

#### 手动补一条临时排班

```http
POST /api/schedules
Authorization: Bearer {token}
Content-Type: application/json

{
  "doctorId": 9206,
  "departmentId": 9103,
  "scheduleDate": "2026-07-07",
  "timeSlot": "MORNING",
  "sourceCount": 8,
  "feeAmount": 120,
  "sourceType": "EXPERT"
}
```

**管理端 UI 建议**：列表每行提供「改号源」「停诊本次」按钮；停诊后列表仍可见（`bookableOnly=false`），状态显示 `CLOSED`。


### 6.3 管理端 API 封装示例（伪代码）

```javascript
// api/schedule.js
const BASE = 'http://127.0.0.1:10010/api'  // 经网关

export function listTemplates(params) {
  return request.get(`${BASE}/schedule-templates`, { params })
}

export function createTemplate(data) {
  return request.post(`${BASE}/schedule-templates`, data)
}

export function generateSchedules(data = { days: 7 }) {
  return request.post(`${BASE}/schedules/generate`, data)
}

export function listSchedules(params) {
  return request.get(`${BASE}/schedules`, { params })
}

export function updateSchedule(id, data) {
  return request.put(`${BASE}/schedules/${id}`, data)
}

export function closeSchedule(id) {
  return request.post(`${BASE}/schedules/${id}/close`)
}
```

```javascript
const baseUrl = 'http://127.0.0.1:10010/api'   // 开发

```

### 7.2 患者端需改的查询

**原写法**（管理端视角）：

```http
GET /api/schedules?departmentId=9101&doctorId=9201&pageNo=1&pageSize=10
```

**患者端应改为**：

```http
GET /api/schedules?bookableOnly=true&departmentId=9101&doctorId=9201&pageNo=1&pageSize=10
Authorization: Bearer {accessToken}
```

`bookableOnly=true` 时后端自动过滤：

- `schedule_date` ∈ [今天, 今天 + advanceDays - 1]
- `status = ENABLED`
- `available_count > 0`

### 7.3 抢号

```javascript
// 只传 scheduleId
```

## 10. 常见问题 FAQ

| 现象 | 原因 | 处理 |
|------|------|------|
| `doctor_schedule_template does not exist` | 未执行 DDL | 跑 `004-schedule-template.sql` |
| 创建模板 500「已有模板」 | 唯一约束 | 换时段或 dayOfWeek，或先 disable 旧模板 |
| generate `createdCount=0` | 实例已存在 | 正常 skipped；换日期或删旧排班 |
| 抢号「号源已满」 | Redis 无库存 | 创建排班时会自动写 Redis；旧数据可手动 SET |
| 抢号「不在预约窗口」 | 超出 7 天 | 患者只能挂窗口内排班 |
| 10023 端口占用 | 服务已启动 | 直接用，不必重复 run |
| 如何暂停某一次排班 | 需停实例非模板 | `POST /api/schedules/{id}/close` |
| 停诊后患者还能挂吗 | CLOSED 不可预约 | `bookableOnly=true` 自动过滤 |
| 停诊后下次 generate | 同日期同时段已存在则 skip | 若要恢复需手动 POST 新建或删旧记录后 regenerate |
